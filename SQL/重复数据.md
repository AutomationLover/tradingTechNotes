## 总体思路

视频里讲的是：**先识别重复，再“在查询里”排除或选一条代表记录**，不直接改表。  
工具组合主要是：`GROUP BY + COUNT + HAVING`、`DISTINCT`、窗口函数 `ROW_NUMBER()`。

---

## 1. 如何识别 duplicate

### 1）按某个字段看“谁重复了”

目标：找出名字重复的员工。

思路：

- `GROUP BY employee_name`
- `COUNT(*)` 统计每个名字出现次数
- `HAVING COUNT(*) > 1` 只保留重复的

示例：

SELECT
  employee_name,
  COUNT(*) AS dup_count
FROM employee_details
GROUP BY employee_name
HAVING COUNT(*) > 1;

扩展：  
- 看 “region + employee_name” 组合是否重复：在 SELECT / GROUP BY 里同时放 `region, employee_name`。  
- 看“整行完全重复”：把所有列都放进 SELECT + GROUP BY，再 `HAVING COUNT(*) > 1`。

---

## 2. 如何排除“完全重复行”

这种最简单：**每列都一样，只是整行被插了多次**。

直接用 `DISTINCT`：

SELECT DISTINCT
  region,
  employee_name,
  salary
FROM employee_details;

效果：  
- 把完全一样的行折叠成 1 行。  
- 适合要“去重后的视图”，不关心重复背后的业务原因时。

---

## 3. 处理“部分重复行”：选一条代表记录（窗口函数）

典型情况：同一个员工有多条记录，但部分字段不同，例如：

- `employee_name` 一样  
- `region` 可能一样或不一样  
- `salary` 不一样（可能代表涨薪／历史记录）

这里不能用 `DISTINCT` 简单去重，需要定义“同一人”的粒度，并选一条“代表行”。

### 3.1 以 employee_name 为唯一键：保留最高工资那条

目标：每个名字只保留一条，选 `salary` 最大的那行。

做法：

1. 用 `ROW_NUMBER()` 按名字分组、按工资排序。
2. 只保留 `row_number = 1` 的那条。

示例：

SELECT
  region,
  employee_name,
  salary
FROM (
  SELECT
    region,
    employee_name,
    salary,
    ROW_NUMBER() OVER (
      PARTITION BY employee_name
      ORDER BY salary DESC
    ) AS rn
  FROM employee_details
) t
WHERE rn = 1;

含义：

- `PARTITION BY employee_name`：每个员工一个窗口。  
- `ORDER BY salary DESC`：工资高的排前面。  
- `rn = 1`：只保留每人最高工资那条。

### 3.2 以 (region, employee_name) 组合为唯一键

假设业务规定：同一个 region + name 是一个“实体”，不同 region 的同名是不同人。  
想要：每个 `(region, employee_name)` 组合只保留一条，选工资最高的那条。

仅需改 `PARTITION BY`：

SELECT
  region,
  employee_name,
  salary
FROM (
  SELECT
    region,
    employee_name,
    salary,
    ROW_NUMBER() OVER (
      PARTITION BY region, employee_name
      ORDER BY salary DESC
    ) AS rn
  FROM employee_details
) t
WHERE rn = 1;

效果：

- `Cat` 在 East / West 各保留一条（各 region 选最高工资）。  
- `Eve` 在 West 有两条时，只保留工资高的一条。

---

## 4. 组合套路总结

1）“先看谁重复了”

- 用 `GROUP BY + COUNT + HAVING COUNT(*) > 1`  
- 决定重复粒度：某一个字段、字段组合、或整行。

2）“简单去重复行”

- 用 `SELECT DISTINCT ... FROM ...`  
- 适用于整行完全相同的重复。

3）“复杂去重复：定义唯一键 + 选一条代表记录”

- 唯一键示例：
  - `employee_name`
  - `(region, employee_name)`
  - `(user_id, event_type)` 等
- 用 `ROW_NUMBER() OVER (PARTITION BY 唯一键 ORDER BY 业务优先级)`  
  - 业务优先级可以是：最新时间、最高金额、最高工资等  
- 外层 `WHERE rn = 1` 保留代表行。

---

## 5. 实战建议

- **先分析 duplicate 的业务含义**：是数据脏？还是合法的历史记录？  
- **粒度要明确**：是按 name，还是按 (region, name)，还是按更多字段。  
- **尽量不要直接删表里数据**：通常通过查询（视图、CTE、子查询）过滤和选代表行就够做分析。  
- 对大表建议先用 `GROUP BY + COUNT` 做 profiling，再决定去重策略。
