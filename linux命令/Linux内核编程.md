## Day 1: Foundations

### What is a Device Driver
- A device driver is software that controls a particular type of hardware connected to the system
- Three interfaces: kernel, hardware, and user (via device files)

### What is a Kernel Module
- **Loadable Kernel Modules (LKM)**: Code that can be loaded/unloaded from the kernel on demand
- **Extension**: `.ko` (Kernel Object)
- **Location**: `/lib/modules/<kernel version>/`
- **Uses**: Device drivers, file systems, system calls, network drivers, TTY line disciplines

**Advantages**: Memory efficiency, no kernel rebuild for new functionality, easier debugging, multi-machine maintenance

**Disadvantages**: Memory overhead for module management, late loading (core functionality must be in base kernel), security considerations

**Configuration**: Requires `CONFIG_MODULES=y`

### Types of Modules
- **In-Source Tree**: Present in Linux kernel source
- **Out-of-Tree**: External modules compiled using kernel build context

### Basic Commands
- `lsmod` – List loaded modules (reads `/proc/modules`)
- `modinfo` – Display module information

### Hello World Module
- **Entry/exit points**: `module_init()` and `module_exit()` macros
- **Licensing**: `MODULE_LICENSE()` (GPL, GPL v2, Dual BSD/GPL, Proprietary, etc.)
- **Headers**: `linux/module.h`, `linux/kernel.h` (for printk log levels)

### Compiling Modules (kbuild)
- `make -C /lib/modules/\`uname -r\`/build M=${PWD} modules`
- `obj-m += modulename.o` – kbuild builds `modulename.ko` from `modulename.c`
- Build artifacts: `modulename.o`, `modulename.mod.c`, `modulename.mod.o`, `modulename.ko`
- `modules.order` – compilation order for multiple modules
- `module.symvers` – external symbols defined in your module

### printf vs printk
- `printk()` – kernel-level logging; writes to kernel buffer
- Format: `printk(KERN_log_priority "message\n");`
- Log levels: EMERG, ALERT, CRIT, ERR, WARNING, NOTICE, INFO, DEBUG

### insmod Process
1. Calls `init_module()` → `sys_init_module()`
2. Permission verification
3. `load_module()`: copy ELF to kernel, verify ELF, symbol resolution
4. Add module to doubly linked list
5. Call `module_init` function

### dmesg
- Kernel logs stored in ring buffer
- Commands: `dmesg -c` (clear after print), `dmesg -C` (clear only), `dmesg -t` (no timestamps), `dmesg -l err,warn`, `dmesg -T` (human-readable time), `dmesg -x` (show log level)

## Day 2: Module Mechanics

### insmod vs modprobe
| insmod | modprobe |
|--------|----------|
| Loads by path | Loads from `/lib/modules/$(uname -r)` only |
| No dependency resolution | Loads dependencies via `depmod` |
| `insmod /path/to/module.ko` | `modprobe modulename` |

- **depmod**: Creates `modules.dep` with dependency info
- Load order: right to left; unload: left to right

### module_init / module_exit Internals
- `module_init(fn)` creates alias: `init_module` → your function
- `module_exit(fn)` creates alias: `cleanup_module` → your function
- Uses GCC `alias` attribute
- `initcall_t` = `int (*)(void)` – init must return int

### Passing Parameters
- **module_param(name, type, perm)** – single parameter
- Types: `charp`, `bool`, `invbool`, `int`, `long`, `short`, `uint`, `ulong`, `ushort`
- Permissions: `S_IRUGO`, `0` (no sysfs), or numeric (e.g., `0644`)
- Usage: `sudo insmod module.ko name="value" loop_count=5`
- View: `cat /sys/modules/<module>/parameters/<param>`
- **Builtin modules**: kernel command line `module.param=value`
- **modprobe**: reads `/etc/modprobe.conf`

### Passing Arrays
- **module_param_array(name, type, nump, perm)**
- Usage: `insmod module.ko param_array=1,2,4`

### String Parameters
- Use single quotes to pass spaces: `insmod module.ko name='"Linux World"'`

### Symbol Table
- **Symbol**: Name for memory (variable or function)
- **Symbol table**: Compiler-generated; kernel has `/boot/System.map-<version>`
- **Export**: `EXPORT_SYMBOL` (any module) vs `EXPORT_SYMBOL_GPL` (GPL only)
- **Verification**: `cat /proc/kallsyms | grep symbol_name`
- `/proc/kallsyms` includes dynamic modules; `System.map` only builtin

### Module Stacking
- New modules using symbols from other modules
- Examples: msdos → fat; lp (printer) → parport
- Use `modprobe` with symlinks in `/lib/modules/$(uname -r)/kernel/` + `depmod -a`

### Vermagic
- Magic string in `.modinfo` section
- Verifies module was compiled for current kernel
- Includes: UTS_RELEASE, SMP, PREEMPT, MODULE_UNLOAD, MODVERSIONS, ARCH, RANDSTRUCT

### THIS_MODULE
- Points to `struct module` for your module
- Generated by kbuild; used with macros like `MODULE_VERSION`

## Day 3: Debugging, Licensing, Build

### Licensing and Tainted Kernel
- **MODULE_LICENSE**: Required; unrecognized = proprietary
- **Tainted kernel**: Unsupported state; some debugging disabled
- Causes: proprietary modules, staging drivers, out-of-tree modules, OOPS, etc.
- Query: `cat /proc/sys/kernel/tainted`
- Decode: `kernel-chktaint` script
- Taint flags: P (proprietary), F (force load), O (out-of-tree), E (unsigned), D (oopsed), etc.

### Module Metadata
- `MODULE_DESCRIPTION`, `MODULE_AUTHOR`, `MODULE_VERSION`
- ELF sections: `objdump --section-headers ./module.ko`
- `.modinfo` section: `objdump --section=.modinfo --full-contents ./module.ko`

### modinfo
- Displays module metadata from `.modinfo` section

### insmod Deep Dive
- Full flow: user space → `init_module()` → `sys_init_module()` → `load_module()` → symbol resolution → `module_init` call

### dump_stack()
- Prints stack trace at call site

### Kernel Panic
- Kernel stops to avoid data loss
- Causes: hardware/software issues, driver bugs, bad RAM
- Reboot: kernel cmdline `panic=N` or `echo N > /proc/sys/kernel/panic`

### OOPS (Oops Message)
- Like segfault in kernel; invalid memory access
- Kernel: kills process, prints debug info, continues (system untrusted)
- OOPS contains: processor status, CPU registers, stack trace, call trace
- Error code bits: Page found/protection, Read/Write, Kernel/User
- Taint flags in output (P, O, E, etc.)

### BUG / WARN Macros
- **BUG_ON(cond)** or **BUG()**: Prints registers, stack trace; process dies
- **WARN_ON(cond)** / **WARN()**: Prints registers, stack trace; continues
- `BUG()` leaves module in use (cannot rmmod)

### KBUILD_MODNAME
- `__stringify(KBUILD_MODNAME)` – module name at compile time

### CFLAGS for Modules
- `CFLAGS_modulename.o := -DDBG` – flags for specific object
- `ccflags-y := -DDEBUG` – flags for all sources in module

### KBUILD_EXTRA_SYMBOLS
- For external modules using symbols from other external modules
- `KBUILD_EXTRA_SYMBOLS := /path/to/Module.symvers`

### Process/Task List
- Kernel uses `struct task_struct` (in `<linux/sched.h>`)
- Stored in circular doubly linked list (task list)
- Fields: process name, PID, state
- States: TASK_RUNNING, TASK_INTERRUPTIBLE, TASK_UNINTERRUPTIBLE, __TASK_TRACED, __TASK_STOPPED

## Day 4: Logging and Debugging

### printk Log Levels
- `KERN_EMERG` (0) through `KERN_DEBUG` (7)
- Defined in `<linux/kern_levels.h>`
- Default: `KERN_DEFAULT` if not specified

### pr_* Macros
- `pr_emerg`, `pr_alert`, `pr_crit`, `pr_err`, `pr_warning`, `pr_notice`, `pr_info`, `pr_debug`
- Wrappers around `printk` with appropriate levels

### pr_debug
- Removed by compiler if `DEBUG` not defined
- Use `CONFIG_DYNAMIC_DEBUG` for runtime control

### Console and printk
- Console: text device (e.g., Ctrl+Alt+F1–F6)
- In graphics mode, kernel messages don't appear in xterm (X server doesn't receive them)
- `cat /proc/sys/kernel/printk` – console_loglevel, default_message_loglevel, etc.
- `echo 8 > /proc/sys/kernel/printk` or `dmesg -n 3` to change levels

### Ring Buffer
- Size: `1 << CONFIG_LOG_BUF_SHIFT`
- Check: `cat /boot/config-\`uname -r\` | grep CONFIG_LOG_BUF_SHIFT`
- Increase: `log_buf_len=4M` on kernel command line
- `dmesg -s 256000` for larger buffer read

### No Floating Point in Kernel
- FPU off for performance; kernel avoids FPU ops
- User space: trap → kernel enables FPU → restore state → return

### printk_ratelimit
- Limits logging rate
- `/proc/sys/kernel/printk_ratelimit_burst` (default 10)
- `/proc/sys/kernel/printk_ratelimit` (seconds to wait after burst)
- Returns 1 (allow) or 0 (block)

### printk_once
- Prints message once, never again

### KERN_CONT
- Prevents newline; allows multi-part messages on one line

### print_hex_dump
- `print_hex_dump(level, prefix_str, prefix_type, rowsize, groupsize, buf, len, ascii)`
- prefix_type: `DUMP_PREFIX_OFFSET`, `DUMP_PREFIX_ADDRESS`, `DUMP_PREFIX_NONE`

### print_hex_dump_bytes
- Shorthand: KERN_DEBUG, rowsize 16, groupsize 1, ASCII included

### Dynamic Debug (CONFIG_DYNAMIC_DEBUG)
- Enable/disable `pr_debug()` and `print_hex_dump_debug()` per callsite
- Mount: `mount -t debugfs none /sys/kernel/debug`
- Control: `echo '<matches> <ops><flags>' > /sys/kernel/debug/dynamic_debug/control`
- Matches: `file`, `func`, `line`, `module` (supports `*`, `?`)
- Ops: `+` add, `-` remove, `=` set
- Flags: `p` (print), `f` (function), `l` (line), `m` (module), `t` (thread ID)
- Boot: `dyndbg="file ec.c +p"` or `module.dyndbg="QUERY"`

## Day 5: Processes, Memory, Kernel Threads

### num_online_cpus
- Count of online CPUs in the system

### Process List (print_tasks)
- Iterate task list; print name, PID, state
- Process states: R (running), S (interruptible), D (uninterruptible), T (stopped)

### current Macro
- Returns pointer to `task_struct` of current process
- Architecture-specific (register or kernel stack)

### find_task
- Finding specific tasks in the task list

### Process Memory Map (VMA)
- `struct mm_struct` – process address space; `current->mm`
- `struct vm_area_struct` – memory region
  - `vm_start`, `vm_end` – linear address interval
  - Linked list in ascending address order
  - `vm_next` – next region

### Kernel Threads
- **Definition**: Linux task running only in kernel mode; not from fork/clone
- **Purpose**: Background kernel operations
- **Examples**: `ksoftirqd`, `kworker`
- **Difference from user thread**: `mm` = NULL (no address space)

### Creating Kernel Threads
- `kthread_create(threadfn, data, name, ...)` – creates but doesn't run
- `wake_up_process(task)` – start thread
- `kthread_run(threadfn, data, name, ...)` – create and run
- `kthread_stop(task)` – stop (blocking)
- **Important**: Check `kthread_should_stop()` in loop; call `kthread_stop()` in `module_exit` or get OOPS

### Parallel Execution
- Multiple kthreads run in parallel (interleaved output demonstrates this)

### smp_processor_id()
- Returns ID of CPU on which code is running
- Use when you have multiple processors

- **Makefile**: `obj-m := module.o` for single module

- **Custom name**: Use `module.ko` from `module.o` via kbuild
- **Clean**: `make -C /lib/modules/$(uname -r)/build M=$(PWD) clean`
