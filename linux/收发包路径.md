# Linux 内核中 `tcp_sendmsg` / `tcp_recvmsg` / `sys_sendto` / `sys_recvfrom` 的大致路径

下面按“从用户态到内核、再到 NIC / socket 队列”的方向，概述这些函数在 Linux 内核中的位置和调用链。细节省略锁、错误处理，偏向网络性能与 eBPF 观测视角。

---

## 1. `tcp_sendmsg` 的路径（TCP 发送）

以应用调用 `send()` / `sendto()` / `write()` 往 TCP socket 写数据为例。

### 1.1 用户态到系统调用入口

典型调用链：

- `send()` → `__libc_send()` → `sys_sendto()` / `__x64_sys_sendto`
- `write()` → `__libc_write()` → `sys_write()` → 内核内部转到 `sock_write_iter` → `sock_sendmsg`

### 1.2 系统调用到 socket 层

简化后的内核调用链：

- `sys_sendto` / `__x64_sys_sendto`  
  - `__sys_sendto`  
    - `sock_sendmsg`（通用 socket 发送入口）  
      - `inet_sendmsg`（AF_INET / AF_INET6 通用入口）  
        - `tcp_sendmsg`

### 1.3 `tcp_sendmsg` 内部关键步骤

`tcp_sendmsg` 大致完成：

- 根据应用传入的 buffer 构造 `sk_buff`：从用户空间拷贝数据或建立 `skb` 链；
- 将 `skb` 挂入 TCP 发送队列（write queue / send queue）；
- 调用 TCP 输出函数发送，大致链路：

- `tcp_sendmsg`  
  - `tcp_push()` / `__tcp_push_pending_frames()`  
    - `tcp_write_xmit()`  
      - `tso_fragment()` / `tcp_transmit_skb()`  
        - `ip_queue_xmit()` → `ip_local_out()`

### 1.4 从 IP 层到设备队列

继续向下：

- `ip_queue_xmit`  
  - `ip_local_out`  
    - `__ip_local_out`  
      - `ip_output`  
        - `ip_finish_output`  
          - `ip_finish_output2`  
            - `dev_queue_xmit(skb)`

在 `dev_queue_xmit` 中，skb 被放入对应网卡的 TX 队列（qdisc/多队列考虑在此），驱动负责 DMA 和物理发送。

> 观测点示例：  
> - 在 `tcp_sendmsg` 上挂 kprobe：统计应用写入 TCP 的行为和耗时。  
> - 在 `dev_queue_xmit` 上挂 kprobe/tracepoint：观测真正进入设备层的发包。

## 2. `tcp_recvmsg` 的路径（TCP 接收）

这里聚焦“从 socket 接收队列到应用 `recv()`”这一段。

### 2.1 NIC 到 socket 队列（简略）

完整 RX 路径（高度简化）：

- NIC 收包 → 中断 / NAPI poll → `netif_receive_skb`  
  - `ip_rcv` / `ip_local_deliver`  
    - `tcp_v4_rcv`  
      - `tcp_rcv_established`  
        - 将 `skb` 挂到 socket 的 `sk_receive_queue`，并唤醒阻塞在 `recv` / `epoll` 上的进程

### 2.2 用户态 `recv()` / `read()` 进入内核

用户态调用：

- `recv()` → `__libc_recv()` → `sys_recvfrom()` / `__x64_sys_recvfrom`
- `read()` → `__libc_read()` → `sys_read()` → 内核内部路由到 socket 的 `sock_read_iter` → `sock_recvmsg`

### 2.3 系统调用到 socket 层

简化调用链：

- `sys_recvfrom` / `__x64_sys_recvfrom`  
  - `__sys_recvfrom`  
    - `sock_recvmsg`（通用 socket 接收入口）  
      - `inet_recvmsg`  
        - `tcp_recvmsg`

### 2.4 `tcp_recvmsg` 内部关键步骤

`tcp_recvmsg` 主要负责：

- 从 `sk_receive_queue` / prequeue / backlog 中取出 `skb`；
- 处理 TCP 报文：校验、序号、重组；
- 将数据拷贝到用户缓冲区（或走零拷路径，视具体实现）；
- 更新 TCP 状态（窗口、ACK 等）。

> 观测点示例：  
> - 在 `tcp_recvmsg` 入口/返回处挂 kprobe：测“socket 队列 → 用户态 buffer”这段耗时。  
> - 配合在 `tcp_v4_rcv` / `netif_receive_skb` 上的打点，分解内核内部延迟。

## 3. `sys_sendto` / `sys_recvfrom` 的位置（通用 socket 系统调用入口）

这两个是系统调用层入口，负责处理用户参数，然后把工作交给相应协议的 `sendmsg` / `recvmsg` 回调。

### 3.1 `sys_sendto` 调用路径

无论是 UDP 还是 TCP，只要用户调用 `sendto()`，大致路径是：

- `sys_sendto` / `__x64_sys_sendto`  
  - `__sys_sendto`  
    - `sock_sendmsg`  
      - `proto_ops->sendmsg(...)`

其中 `proto_ops->sendmsg` 根据 socket 类型不同而不同：

- TCP socket：`tcp_sendmsg`  
- UDP socket：`udp_sendmsg`  
- RAW socket：`raw_sendmsg` 等

**`sys_sendto` 的职责可以概括为：**

> 把用户态传进来的参数（buffer、长度、地址）封装成一个 `msghdr` + socket，然后调用具体协议的 `sendmsg` 实现。

### 3.2 `sys_recvfrom` 调用路径

同理，`recvfrom()` 的内核路径为：

- `sys_recvfrom` / `__x64_sys_recvfrom`  
  - `__sys_recvfrom`  
    - `sock_recvmsg`  
      - `proto_ops->recvmsg(...)`

- TCP socket：`tcp_recvmsg`  
- UDP socket：`udp_recvmsg`  
- 其它协议：各自的 `recvmsg` 实现

**`sys_recvfrom` 的职责是：**

> 处理用户态参数 → 找到对应 socket → 调用具体协议的 `recvmsg` 回调，从 socket 接收队列读数据，再拷贝回用户态。

## 4. 汇总：实用心智图

面向网络性能调优 / eBPF 观测，可以这样记：

### 4.1 TCP 发送路径（用户态 → NIC）

- 用户态 `send` / `write`  
  - `sys_sendto` / `sys_write`  
    - `sock_sendmsg`  
      - `tcp_sendmsg`  
        - `tcp_write_xmit` / `tcp_transmit_skb`  
          - `ip_queue_xmit` / `ip_local_out`  
            - `dev_queue_xmit`  
              - 驱动 → NIC TX ring → 物理线

### 4.2 TCP 接收路径（NIC → 用户态）

- NIC RX → NAPI poll → `netif_receive_skb`  
  - `ip_rcv` / `ip_local_deliver`  
    - `tcp_v4_rcv` / `tcp_rcv_established`  
      - `skb` 进入 socket `sk_receive_queue` + 唤醒进程  

- 用户态 `recv` / `read`  
  - `sys_recvfrom` / `sys_read`  
    - `sock_recvmsg`  
      - `tcp_recvmsg` 从接收队列取出数据 → 拷贝到用户 buffer

### 4.3 UDP 与 `sys_sendto` / `sys_recvfrom`

- UDP 发送：`sys_sendto` → `sock_sendmsg` → `udp_sendmsg` → IP → `dev_queue_xmit` → NIC  
- UDP 接收：NIC → IP → `udp_rcv` → socket 队列 → `sys_recvfrom` → `sock_recvmsg` → `udp_recvmsg`

`sys_sendto` / `sys_recvfrom` 本身不做协议逻辑，是系统调用层的“入口 + 分发”，真正的网络行为在 `tcp_*` / `udp_*` / `raw_*` 等函数中完成。
